História de usuário:

No desenvolvimento de software e no gerenciamento de produtos, uma história de usuário é uma descrição que consiste em uma ou mais sentenças na linguagem cotidiana ou de negócios do usuário final ou usuário de um sistema que captura o que um usuário faz ou precisa fazer como parte de seu função de trabalho.

Regras:
  • Não olhe para o exemplos de códigos de outros projetos. Descobrir por si mesmo.
  • Você pode usar as bibliotecas ou APIs que você precisa.
  • Como usuário, eu posso acessar todo o conteúdo da página web carteira apenas rolando.
  • Como usuário clicar botões diferentes levarão a diferentes páginas de mídia social da carteira do criador
  • Pode-se ver imagens em miniatura de diferentes projetos da carteira criador construiu.
  • Pode-se navegar para diferentes secções da página Web, clicando nos botões na barra de navegação.

Trabalho de conclusão de curso

Sistema de manutenção com a pilha de tecnologias MEAN e Micro controladores.

É o que estou trabalhando no momento, este trabalho explora deficiências no sistema de bicicletas compartilhadas Bike Rio e tem como objetivo o desenvolvimento de um sistema para apoiar o gerenciamento de serviços de manutenção de bicicletas implementado com o uso de uma arquitetura de software distribuída e orientada a serviço.

Camadas e Tecnologias :
  • Iot- Aplicativo embarcado: Raspberry e arduino (js johnny-five).
  • Disp Movel- App multiplataforma: phonegap/cordova e jquery mobile.
  • Servidor- restfull : node.js, express, pagina SPA e rotas(rest/json).
  • Aplicaçao Web - angular.js , jquery e APIs
  • Banco de Dados - com Mongodb e ferramenta de modelagem mongoose
